# Doe 语言 1.0 规范

## 第一部分：概述与类型系统

### 1.1 语言概述

Doe（豆语言）是一门静态类型、编译型语言，支持**即时编译（JIT）**和**脚本执行**两种后端。语法继承 Cyber 语言，在此基础上进行简化和调整。

**设计哲学**：
- 简洁优先：减少关键字，统一概念
- 显式优于隐式：定义方决定语义，调用方不猜测
- 实用主义：覆盖 95% 场景，不追求完备

**移除的 Cyber 特性**：
- `while` 关键字（`for` 统一所有循环）
- `switch` 关键字（`match` 统一所有分支）
- `cgen` 后端（C 代码生成）
- `LLVM IR` 后端

### 1.2 标识符命名规则

| 类别 | 命名规范 | 强制？ | 示例 |
|------|---------|--------|------|
| 类型名（含 enum/union 子类型） | CamelCase，首字母大写 | **是** | `Animal`, `HttpStatus`, `Ok`, `Some` |
| 变量、函数、方法、属性 | snake_case，全小写 | **是** | `user_name`, `set_name`, `port` |
| 常量 | SNAKE_CASE，全大写 | 建议 | `MAX_SIZE`, `DEFAULT_PORT` |
| 私有标识符 | `_` 前缀 + snake_case | **是** | `_value`, `_helper` |

### 1.3 基础类型（值类型）

| 类型 | 位宽 | 有符号 | 范围/说明 |
|------|------|--------|----------|
| `byte` | 8 | 无符号 | 0–255 |
| `twine` | 16 | 无符号 | 0–65535，端口号范围 |
| `rune` | 32 | 有符号 | Unicode 标量值 |
| `int` | 64 | 有符号 | -9223372036854775808 至 9223372036854775807 |
| `float` | 64 | 有符号 | IEEE 754 双精度浮点 |
| `bool` | 1 | — | `true` / `false` |
| `str` | — | — | 不可变 UTF-8 字符串，值类型 |

### 1.4 复合类型（引用类型）

| 类型 | 说明 |
|------|------|
| `list[T]` | 动态数组，可变 |
| `dict[K, V]` | 键值映射，K 需实现 `Comparable` 接口 |
| `class` | 自定义引用类型（聚类） |
| `enum` | 整数枚举 |
| `union` | 命名变体联合 |
| `trait` | 接口契约（特质） |

### 1.5 特殊类型

| 类型 | 说明 |
|------|------|
| `none` | 空值，表示无结果 |
| `error` | 错误类型，包含 `none` 和具体错误子类型 |
| `any` | 顶层类型，包含所有类型 |
| `some` | 运行时约束：非 `none` 且非 `error` |

### 1.6 类型修饰符

| 修饰符 | 展开形式 | 说明 |
|--------|---------|------|
| `T?` | `Option[T]` | 可选类型，可能为 `none` |
| `T!` | `Result[T]` | 结果类型，可能为 `error` |

**规则**：
- `?` 和 `!` **互斥**，不组合
- `T!` 中的 `error` 本身包含 `none`，但 `-!` 操作符**不处理 `none`**（`none` 不是错误，属于正常语义）

### 1.7 字符串处理

`str` 是不可变值类型。修改需显式转换为 `list[rune]`：

```doe
let s = "hello"               # str，不可变
let chars = s.runes()         # list[rune]，可变
chars.append('!')
let s2 = chars.toStr()        # 转回 str
```
